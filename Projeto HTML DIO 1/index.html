Breve Histórico do HTML

O HTML foi criado pelo Sir Tim Berners-Lee nos fins de 1991, mas não foi lançado oficialmente até 1995, quando a versão 2.0 saiu ao público. Já nessa época, o HTML se destacava como o padrão de linguagem markup para o design e criação de sites para a web, que praticamente surgiu junto com ele. Já em 1997, o HTML 3.0 veio com grandes novidades para os desenvolvedores, adicionando funcionalidades ao desenvolvimento web. Porém essas novas ferramentas também começaram a diminuir a velocidade dos navegadores ao carregar as páginas. Em 1999, foi lançado o HTML 4.0, que permaneceu laragamente usado e distribuído, inclusive após o lançamento da versão 5.0, em 2012, que é a versão atual.

Definição e conceito de HTML

HTML significa HyperText Markup Language, ou, traduzindo para o português livre, seria lingugagem de marcação de hipertexto, mas o que significa isso? Vamos quebrar em partes

Linguagem de marcação é "um conjunto de sinais e códigos aplicados a um texto ou a dados para para definir formatos, maneiras de exibição e padrões", ou seja, é uma forma de mostrar para o navegador como o texto deve ser exibido, cada palavra e bloco de texto, imagem, etc.

Já os hipertextos são simplesmente os links que conectam as páginas umas às outras, seja dentro de um mesmo diretório, ou através da web. Essa forma de unir páginas foi essencial para o surgimento e desenvolvimento da internet como a conhecemos, pois permitiu que a troca de informações crescesse de forma exponencial.

Assim, o HTML nada mais é do que um formato de texto padronizado, um padrão de escrita que é interpretado pelos navegadores, possibilitando a sua exibição para o usuário final da forma que o desenvolvedor deseja. Alguns costumam chamar o HTML de o "esqueleto" da web, pois é em cima dele que as outras partes da programação para web são criadas, como o CSS e o JavaScript.

Olhando para o futuro: o que esperar do HTML?

Como dito acima, o HTML funciona como o esqueleto da web, então ele precisa estar sempre atualizado com as últimas demandas e novidades que os consumidores de conteúdo e desenvolvedores estão trazendo. Nesse sentido, a WHATWG, que é o órgão responsável por definir as especificações do padrão HTML, está trazendo algumas atualizações em elementos nativos, de forma que tenham mais interatividade, incrementando algumas funções que já estão presentes no JavaScript, por exemplo.

Além disso, há planos de tornar o HTML mais rápido com renderiação do lado do servidor. Isso seria feito por meio de Web-sockets, o que permitirá performance sólida, rápido desenvolvimento de funcionalidades, e designs responsivos sem a necessidade de construir duas aplicações separadas para isso.

Há que se levar em conta o caminho em direção ao desenvolvimento cloud-native, e a grande atenção que está se dando para as web apps, que estão trazendo serviços cada vez mais completos, requerendo uma modelagem web firme e capaz de entregar numerosas funcionalidades.
